# Meallens Exp (Data Analyst)

At Meallens, I worked with CRM and ERP systems.

My main duty was to troubleshoot application and data issues, support business users, and maintain Power BI reports and datasets.

I solved problems like incorrect or missing data, reporting errors, and recurring system issues. 

I used SQL for data analysis and fixes, Power BI for reporting, and ticketing tools for support tracking.

# Project 1: ERP Data Issue Resolution System

In this project, I designed a SQL-based system to detect and fix data issues in an ERP environment.

The problem was that data inconsistencies and integration errors were hard to identify manually, which caused delays in reporting and operations.

I used SQL Server to query and validate data across tables, and created python scripts to detect missing, duplicate, or mismatched records. I also built automated checks to flag issues early and used stored procedures for controlled data corrections with rollback safety.

The method I followed was: identify data rules, build validation queries, detect anomalies, fix data using controlled SQL scripts, and document each step for repeatability.

The result was a 50% reduction in manual debugging effort and faster identification of ERP data issues, which improved data reliability for reporting and operations teams.


# Project 2: Power BI Operations Support Dashboard
In this project, I build a Power BI dashboard to support operations teams in tracking incidents and system performance.

The problem was that managers had no single view of ticket status, SLA (Service Level Agreement) performance, and system issues. They were manually checking multiple reports, which was slow and inconsistent.

I used SQL to extract and clean data from support and operational systems, and Power BI with DAX to build KPIs (Key Performance Indicator) like SLA (Service Level Agreement) compliance, ticket volume, and resolution time. I created a data model with relationships between tickets, users, and time data.

The method I followed was: collect requirements from users, design SQL datasets, build a data model, create DAX measures, and publish the dashboard in Power BI Service with scheduled refresh.

The result was improved visibility for managers, faster decision-making, and a 40% increase in dashboard usage because users relied on it instead of manual reports.

# Zapier
I have worked on an Automated Lead Capture and Follow-Up System project using Zapier. The goal was to reduce manual work and make sure every new lead gets a quick response.

When a customer fills out a form on the Google Forms, Zapier automatically starts the workflow. First, the lead information is saved in Google Sheets.

Then, a notification is sent to the Slack channel so the team know a new lead has arrived. After that, the system automatically sends a personalized welcome email to the customer through Gmail. If a lead meets certain criteria, Zapier automatically assigns that lead to the high-priority for faster follow-up.
